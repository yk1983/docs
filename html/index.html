a tag

정의 :

a 요소는 한 페이지에서 다른 페이지로 연결하는 데 사용되는 하이퍼 링크를 정의합니다.
a 요소의 가장 중요한 속성은 링크의 목적지를 나타내는 href 속성입니다.
기본적으로 링크는 모든 브라우저에서 다음과 같이 나타납니다.

  • 방문하지 않은 링크에는 밑줄이 그어져 파란색으로 표시됩니다.
  • 방문한 링크에 밑줄이 그어지고 자주색입니다.
  • 활성 링크에는 밑줄이 그어지고 빨간색으로 표시됩니다.

설명 :

a 요소는 문단, 목록, 테이블뿐만 아니라 섹션 전체를 감쌀 수도 있습니다.
a 요소안에 버튼이나 링크등 인터렉티브한 요소가 포함되지 않도록 합니다.

링크의 상태 :

스타일을 정의할 때 링크 상태들은 순서대로 정의 해야 의도한 대로 스타일을 적용할 수 있다. active를 hover보다 먼저 정의 했다면 hover가 active를 오버라이드 하여 active상태를 볼 수가 없게 된다.

  • link : 기본적인 링크 상태로 아직 방문한적이 없거나 아무런 행동도 하기전인 상태
  • visited : 방문한 적이 있는 링크의 상태
  • hover : 마우스 커서가 링크위에 올라가 있는 상태
  • focus : 링크에 키보드등을 이용해서 포커스가 위치해있는 상태
  • active : 마우스를 눌렀다가 놓는 동안의 상태

지원 브라우저 :

속성 :

Attribute Description
charset 링크 된 문서의 문자 집합을 지정합니다. HTML5에서는 지원되지 않습니다.
coords 링크의 좌표를 지정합니다. HTML5에서는 지원되지 않습니다.
download 사용자가 하이퍼 링크를 클릭 할 때 대상을 다운로드하도록 지정합니다.
href URL 링크가 연결되는 페이지의 URL을 지정합니다.
hreflang 링크 된 문서의 언어를 지정합니다.
media 링크 된 문서가 최적화되어있는 미디어 / 장치를 지정합니다.
name 앵커의 이름을 지정합니다. HTML5에서는 지원되지 않습니다. 대신 global id 속성을 사용하십시오.
ping 링크를 따라 가면 본문 핑이있는 게시물 요청이 브라우저에서 백그라운드로 전송되는 URL의 공백으로 구분 된 목록을 지정합니다. 일반적으로 추적에 사용됩니다.
rel 현재 문서와 링크 된 문서 간의 관계를 지정합니다.
rev 링크 된 문서와 현재 문서 간의 관계를 지정합니다. HTML5에서는 지원되지 않습니다.
shape 링크 모양을 지정합니다. HTML5에서는 지원되지 않습니다.
target 링크 된 문서를 열 위치를 지정합니다. (_blank, _parent, _self, _top, framename)
type 링크 된 문서의 미디어 유형을 지정합니다.

abbr tag | acronym tag

정의 :

abbr요소는 텍스트를 생략어(abbreviated form) 즉, "Mr.", "Dec", "ASAP", "ATM"과 같은 약어 또는 머리 글자를 정의합니다.
acronym요소는 텍스트를 두문자어(initial letter)로 지정합니다. HTML5에서 지원되지 않습니다. 대신 abbr요소를 사용합니다.

Tip

약어를 마크 업하면 브라우저, 번역 시스템 및 검색 엔진에 유용한 정보를 제공 할 수 있습니다. 약어와 머리 글자 어는 모두 다른 것의 축약 된 버전입니다. 둘 다 종종 일련의 문자로 표현됩니다.

지원 브라우저 :

속성 :

Attribute Description
title 약어 전체의 의미를 지정한다.

설명 :

요소위에 마우스를 올리면 title속성의 내용이 툴팁으로 보여진다.

생략어(abbr)과 두 문자어(acronym)

생략어는 스펠링을 한자씩 발음하여 읽어주고, 두문자어는 한 단어처럼 발음하여 읽어주는 차이가 있다.

예제 :

address tag

정의 :

address요소는 문서 또는 기사의 작성자 / 소유자에 대한 연락처 정보를 정의합니다.
address요소가 <body> 요소 안에 있으면 문서의 연락처 정보를 나타냅니다.
address요소가 <article> 요소 안에 있으면 해당 article의 연락처 정보를 나타냅니다.
address요소의 텍스트는 대개 이탤릭체로 표시 됩니다. 대부분의 브라우저는 주소 요소 앞뒤에 줄 바꿈을 추가합니다.

지원 브라우저 :

Note

연락처 정보의 일부가 아닌 경우 <address> 태그는 우편 주소를 설명하는데 사용하지 않도록 합니다.

Tip

<address> 요소는 일반적으로 다른 정보와 함께 <footer> 요소에 포함됩니다
